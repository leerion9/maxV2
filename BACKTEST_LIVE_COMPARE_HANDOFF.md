- **거래대금 300% (당일 EOD)** 는 룩어헤드가 크다 → 화려한 수치의 상당 부분은 **장 마감 후** 정보.

### 3-2. 매수 신호 (`VolatilityBreakoutStrategy.on_quote`)

**A → B 순서** (동시 AND 아님):

| 단계 | 조건 |
|------|------|
| 돌파가 산출 | `breakout_price = 시가 + (전일고−전일저) × breakout_k` (K=0.7) |
| **A** | 장중 **누적 거래량** ≥ **5일 평균 일 거래량** (×**100%**, 1배) |
| **B** | **현재가** ≥ `breakout_price` |
| 스킵 규칙 | **첫 호가**에서 A·B **동시 충족** → 당일 해당 종목 **진입 안 함** (갭업 추격 방지) |

## 6. 장중 거래대금 프록시 — **진행 안 함** (2026-07-04 합의)

### 배경

일봉만으로 실매매의 「돌파 시점 장중 누적 거래량/거래대금」을 재현하려면 **프록시(가정 모형)** 가 필요했음.

### 검토한 후보 (백테스트 관점)

| 후보 | 요약 | 결론 |
|------|------|------|
| A. EOD 당일 거래대금 300% | 현재 baseline | 이미 검증. 룩어헤드 **상한** |
| B. 전일 거래대금 300% (PIT) | vol_lag=1 | 이미 검증(A1). 실전과 **다른 신호** |
| C. 가격경로 비례 누적 | `V* = Volume × (target−Open)/(High−Open)` | **유일한 일봉 장중 추정 시도**이나 가정 겹침 → **민감도용**, 실전 재현 불가 |
| D. EOD 거래량 배수 | 거래대금→거래량만 교체 | EOD 룩어헤드 동일. 프록시 **신규 가치 없음** (같은 종목 내 지표 교체는 가격 변동 시 신호는 달라지나, §7-4 목적과 무관) |

### 사용자 결정

- **프록시(C 포함) 백테스트는 진행하지 않음.**  
- 이유: 분봉·틱 없음 + 시점·누적 추정의 이중 가정 → **실전 정합 백테스트로 쓰기 어려움**.

### 향후 실매매 정합 시 대안

- 백테스트: **baseline(EOD 300%)** 를 “낙관 상한” 참고로만 사용.  
- 실매매: 현행 **장중 누적 거래량 100% + A→B** 유지 또는 API로 **장중 누적 거래대금** 직접 사용 검토.  
- 분봉 데이터 확보 시 그때 **진짜 장중 조건** 백테스트 재개.
